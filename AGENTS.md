## What is Recall?

Recall is a desktop app that helps you fill in your Harvest time tracking by building a timeline of what you actually did on a given day. It pulls activity data from multiple sources ‚Äî GitHub, Google Workspace, local git repos, JIRA, Zulip ‚Äî and presents them on a single timeline so you can quickly see what you worked on and when.

The target user is a developer at Reload (the company) who uses Harvest for time tracking.

---

## Data sources

Each source provides timestamped activity events that are merged into a day timeline.

| Source | What it captures | Auth / access |
|--------|-----------------|---------------|
| **GitHub** | PRs opened/reviewed/merged, commits pushed, issues commented on | Personal access token or GitHub CLI (`gh`) |
| **Google Calendar** | Meetings and events you attended | Google OAuth (Calendar API) |
| **Gmail** | Emails you **sent** or **replied to** (not all received mail) | Google OAuth (Gmail API) |
| **Google Drive** | Docs/Sheets/Slides you **edited** | Google OAuth (Drive Activity API) |
| **Local git** | Commits across all local repos, by your git author name | Shell: `git log --all --since=... --author=...` |
| **JIRA** | Tickets you commented on, transitioned, or were assigned to | JIRA API token |
| **Zulip** | Messages you **sent** | Zulip API key |

Potential future sources: Browser history, terminal/shell history, Harvest (to show already-logged entries alongside the timeline).

## Project structure

| Path | Purpose |
|------|--------|
| `src/` | Frontend: SvelteKit routes, components, app shell |
| `src-tauri/` | Tauri/Rust backend: commands, DB, app state |
| `static/` | Static assets |
| `build/` | Vite build output (Tauri uses this as `frontendDist`; git-ignored) |

**Frontend (`src/`):**

- `src/app.html` ‚Äî HTML shell with dark mode detection
- `src/bindings.ts` ‚Äî **Generated by tauri-specta from Rust commands. Do not edit.** Re-generated in debug builds via `lib.rs`.
- `src/components/Main.svelte` ‚Äî Main tab content (will become the timeline view)
- `src/components/Settings.svelte` ‚Äî Settings tab (theme switcher + source credentials)
- `src/lib/components/ui/` ‚Äî shadcn-svelte UI primitives (button, card, tabs, select, toggle, sonner, etc.)
- `src/lib/utils.ts` ‚Äî `cn()` helper and utility types
- `src/routes/` ‚Äî SvelteKit routes: `+layout.svelte`, `+page.svelte` (tab shell), `layout.css` (retro theme)

**Backend (`src-tauri/`):**

- `src-tauri/src/lib.rs` ‚Äî Tauri entry, command registration, tauri-specta export
- `src-tauri/src/commands/settings.rs` ‚Äî Language get/set + generic key-value helpers (`save_val`, `get_val`)
- `src-tauri/src/db.rs` ‚Äî SQLite init (single `settings` table: key, value, updated_at)
- `src-tauri/src/state.rs` ‚Äî AppState with DB mutex
- `src-tauri/capabilities/` ‚Äî Tauri permissions
- `src-tauri/icons/` ‚Äî App icons

## Workflows

### Install

- From repo root: `npm install`. Rust: `cargo` in `src-tauri/` (no extra install step).

### Dev

- Frontend only: `npm run dev` ‚Üí Vite at **http://localhost:1420**
- Full app: `npm run tauri dev` (runs `npm run dev` then opens Tauri window).

### Build

- Frontend: `npm run build` ‚Üí output in `build/`
- Desktop: `npm run tauri build` (runs `npm run build` then Tauri bundle).

### Typecheck & format

- `npm run check` ‚Äî SvelteKit sync + svelte-check
- `npm run format` ‚Äî Prettier on `src/` only

## Conventions

- **Do not edit `src/bindings.ts`** ‚Äî It is generated by tauri-specta in debug builds. Add or change Tauri commands in `src-tauri/src/commands/` and register them in `lib.rs`.
- **Imports** ‚Äî Use `$lib` for lib (SvelteKit default); `@/*` ‚Üí `./lib/*`. Frontend imports commands and types from `../bindings`.
- **Svelte** ‚Äî PascalCase for main components; `src/lib/components/ui/` uses kebab-case (e.g. `loader.svelte`).
- **Rust** ‚Äî snake_case for modules and command names; commands exposed via tauri-specta in `lib.rs`.
- **Settings storage** ‚Äî Generic key-value store in SQLite (`settings` table). Use `save_val`/`get_val` in `commands/settings.rs` for new settings. Source credentials (tokens, API keys) are stored here.
- **Timeline events** ‚Äî Each data source should produce a common event shape (timestamp, source name, summary text, optional URL) so the frontend can render a unified timeline.
